# Estándar: política de Pull Request

Todo cambio implementado requiere PR antes de integrarse.

## El PR debe incluir

- Objetivo y alcance.
- Resumen de cambios.
- Pruebas ejecutadas.
- Riesgos y aprobaciones requeridas.
- Confirmación de que `main` no fue modificada directamente.

Miguel conserva la aprobación final en cambios críticos.
