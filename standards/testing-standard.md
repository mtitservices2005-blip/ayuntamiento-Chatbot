# Estándar: pruebas

Toda implementación debe incluir pruebas proporcionales al riesgo.

## Mínimos

- Documentación: revisar rutas, nombres y consistencia.
- Código: ejecutar pruebas, lint o validaciones disponibles.
- Seguridad: validar permisos, datos sensibles y límites de exposición.

Si una prueba no puede ejecutarse, debe documentarse la razón.
