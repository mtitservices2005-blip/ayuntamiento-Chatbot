# Estándar: flujo Git

1. Confirmar que no se trabaja en `main`.
2. Crear una rama descriptiva por misión.
3. Implementar cambios pequeños.
4. Ejecutar verificaciones proporcionales.
5. Crear commit descriptivo.
6. Abrir Pull Request para revisión.

Ningún agente modifica `main` directamente ni hace merge sin aprobación humana.
