# Checklist de revisión

- [ ] La rama no es `main`.
- [ ] El alcance está claro.
- [ ] Los commits son descriptivos.
- [ ] Las pruebas están documentadas.
- [ ] No hay secretos ni datos sensibles expuestos.
- [ ] Producción, costos, secretos, eliminación de datos y despliegues tienen aprobación humana si aplican.
- [ ] El PR resume riesgos y limitaciones.
